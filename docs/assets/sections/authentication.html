Authentication

Overview

  • Authentication inside Wakanda Server is handled by the directory
  • The directory offers a default authentication mechanism but gives the possibility to completely override it
  • A default REST API gives access to authentication and session management
  • An SSJS API is available for different kinds of authentication/session operations
  • Multiple authentication schemes can coexist on the same application

Client Side API

The Wakanda JavaScript Client can be used to do all authentication operations. All you need to do is load the connector on your web page and then use its methods to authenticate users.

The Wakanda JavaScript Client is by default included in your project files.

SSJS API

An SSJS API is available through the globally exposed object directory to handle all kind of authentication and session management operations. The Directory interface documentation gives more details about the available methods and properties of this object.

Default REST API

By default, Wakanda Server exposes a REST API so you can handle common directory operations.

  • http://[servername]:[port]/rest/$directory/currentUser
  • http://[servername]:[port]/rest/$directory/currentUserBelongsTo
  • http://[servername]:[port]/rest/$directory/login
  • http://[servername]:[port]/rest/$directory/logout
