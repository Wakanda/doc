Shared Worker

Why

  • You need to execute some code in background.
  • You need to do it in a non blocking way.

What is it

  • A shared worker is running in its own thread.
  • A shared worker runs asynchronously.
  • A shared worker provides an interface for messaging.
  • A shared worker runs V8 4.2.

How to

Inside your Wakanda backend project is where you usually use a shared worker.

It is composed of:

  • A worker running in its own thread.
  • A proxy that helps you to communicate with the worker.
