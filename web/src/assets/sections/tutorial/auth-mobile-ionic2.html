Mobile Login Interface

In this section we create a simple login interface using ionic 2 !
This section requires basic understanding of Ionic 2, if you are not familiar with it we would recommend checking the official Documentation, but if you are a “learn by doing” person this can be a nice introduction.

Let's get our hands dirty !

You can keep building the application without pausing to recompile or refresh the browser if you clicked the Run button with at least backend and mobile checked.

This mobile template is based on the ready made up Tabs template. But we won't be needing all the generated page. Go ahead and delete everything except home page.

Inside the app.module.ts remove all the imports of deleted files. For app.component.ts replace the import of TabsPage by HomePage then set HomePage as rootPage.

Your files should look like this :

When your browser updates, everything should work as expected.

Inside the home page we will be manipulating our todos, but for now let’s start with the login interface.

To create a new page, we make use of the New Framework Element feature.It's based on Ionic CLI that contains the basic commands to get up and running with an ionic 2 project.

Next we need to make a couple of changes to see our interface in action:

  • Add the new login page to the AppModule.
  • Inside app.component.ts change rootPage from home page to login page.
  • Inside the login.ts file import Wakanda service, add it to the providers array then assign it to a public variable.

Login page consists of two input elements that we bind to the name and the password properties, then we check if the user exists in the directory AND belongs to the group todoUsers thanks to wakanda-javascript-client methods : login() and getCurrentUserBelongsTo.

If everything went fine we redirect the user to the home page, otherwise we alert an error message.

You may want to create a service to hold your authentication logic. The services referred as Providers and they will be located in Providers folder.
