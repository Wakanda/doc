Web App Routing

This section requires basic understanding of angular 4, if you are not familiar with it we would recommend checking the official Documentation, but if you are a “learn by doing” person this can be a nice introduction.
In this section we will see how to:
  • Configure routing for our Angular App
  • Add new components and associate them to new routes

Let's get our hands dirty !

Since our application will need two views, the first for login and the second for the Todos, let's create those two components first.

To do so, we make use of the New Framework Element feature that is based on Angular CLI. The command line tool to create angular apps.

Router configuration will happen in our application's module file.

In ./src/app/app.module.ts we need to:

  • Import the Router Module. ```javascript import {{'{'}} RouterModule {{'}'}} from '@angular/router'; ```
  • Add it to AppModule imports array and pass routes configuration.
    • ```javascript imports: [ BrowserModule, FormsModule, HttpModule, RouterModule.forRoot([{{'{'}} path: 'todos', component: TodoComponent {{'}'}}, {{'{'}} path: 'login', component: LoginComponent {{'}'}}, {{'{'}} path: '', redirectTo: '/login', pathMatch: 'full' {{'}'}}]) ]) ```

  • Indicate to the router where the current view should be displayed by adding the router-outlet element.
